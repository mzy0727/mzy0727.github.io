CMake快速入门

什么是CMake

CMake是一个跨平台的安装（编译）工具，可以使用简单的语法来描述所有平台的安装（编译过程）。主要是编写CMakeLists.txt文件，然后使用cmake命令将CMakeLists.txt文件转化为make所需要的makefile文件，最后用make命令编译源码生成可执行程序和共享库。

构建项目

  • 内部构建：直接在源文件目录构建项目，会导致临时文件和源代码放在一起，不好清理。
  • 外部构建：创建一个可以位于文件系统任何位置的构建文件夹。所有临时构建和目标文件都位于此目录中，保持源代码树的整洁。

基础知识

最低版本要求

每个CMakeLists.txt的第一行都需要

从CMake 3.12开始，版本号可以声明为一个范围：

  1. if(${CMAKE_VERSION} VERSION_LESS 3.12):
    • 这是一个条件语句，用于检查当前CMake版本是否低于3.12。
    • 如果当前CMake版本小于3.12，下面的代码块将执行。
  2. cmake_policy(VERSION ${CMAKE_MAJOR_VERSION}.${CMAKE_MINOR_VERSION}):
    • 如果当前CMake版本小于3.12，这行代码将设置CMake策略的版本。
    • CMAKE_MAJOR_VERSION 表示主版本号，CMAKE_MINOR_VERSION 表示次版本号。
    • 这样的设置可能是为了确保在较旧的CMake版本上使用较新的策略，以提供更好的兼容性。

设置一个项目

用于定义项目的基本信息的命令

  • project(MyProject VERSION 1.0:
    • project 命令用于定义一个CMake项目。
    • MyProject 是项目的名称，您可以将其替换为您项目的实际名称。
    • VERSION 1.0 指定了项目的版本号。这是一个可选的参数，可以根据您的需要设置。在这里，版本号被设置为1.0。
  • DESCRIPTION "Very nice project":
    • DESCRIPTION 用于提供对项目的简要描述。
    • 在这里，项目被描述为 “Very nice project”。您可以根据实际情况修改描述。
  • LANGUAGES CXX:
    • LANGUAGES 用于指定项目所使用的编程语言。
    • CXX 表示C++，即项目将使用C++语言。如果您的项目使用其他语言，可以相应地更改此部分。

包含头文件

指定一个目标的头文件包含目录

target_include_directories 为目标添加了一个目录。 PUBLIC 对于一个二进制目标没有什么含义；但对于库来说，它让 CMake 知道，任何链接到这个目标的目标也必须包含这个目录。其他选项还有 PRIVATE（只影响当前目标，不影响依赖），以及 INTERFACE（只影响依赖）。

  • include_directories 用于全局指定整个项目的头文件搜索路径。
  • target_include_directories 用于目标特定的头文件包含目录设置，提供更细粒度的控制，并使用 PRIVATE、PUBLIC 和 INTERFACE 来指定目录的使用属性。

搜索库文件

  • link_directories 用于全局指定整个项目的库文件搜索路径。
  • target_link_directories 用于目标特定的库文件链接目录设置，提供更细粒度的控制，并使用 PRIVATE、PUBLIC 和 INTERFACE 来指定目录的使用属性。

链接库文件

  • target_link_libraries 用于指定目标（可执行文件、静态库或共享库）所需链接的库，具有更细粒度的控制。
  • link_libraries 用于在项目范围内全局设置要链接到所有目标的库，适用于整个项目的通用库。

如果是链接第三方库文件，需要根据参数指定的路径找到库文件

设置变量

声明一个本地变量

变量名通常全部用大写，变量值跟在其后。命令通常是小写。可以通过 ${} 来解析一个变量，例如 ${MY_VARIABLE}

向指定目标添加源文件

target_sources 是一个CMake命令，用于向指定的目标添加源文件。这可以用于向可执行文件、静态库或共享库添加源文件，而无需直接在 add_executable 或 add_library 命令中列举所有源文件。target_sources 允许将源文件的列表分开，从而更好地组织和管理源代码。

  • target_name 是目标的名称，可以是可执行文件、静态库或共享库。
  • PRIVATE|PUBLIC|INTERFACE 用于指定源文件的使用属性。
  • source1 [source2 ...] 是源文件的列表，可以是相对或绝对路径。

日志

(无) ：重要消息
STATUS ：非重要消息
WARNING：CMake 警告, 会继续执行
AUTHOR_WARNING：CMake 警告 (dev), 会继续执行
SEND_ERROR：CMake 错误, 继续执行，但是会跳过生成的步骤
FATAL_ERROR：CMake 错误, 终止所有处理过程
CMake的命令行工具会在stdout上显示STATUS消息，在stderr上显示其他所有消息。CMake的GUI会在它的log区域显示所有消息。

CMake警告和错误消息的文本显示使用的是一种简单的标记语言。文本没有缩进，超过长度的行会回卷，段落之间以新行做为分隔符。
